Vue 初体验

⼀、前端渐进式框架 Vue 基本概念

1. 前端响应式框架 Vue 简介

  • Vue (读⾳ /vjuː/，类似于 view)
  • Vue 的官⽅⽹站
    • https://cn.vuejs.org/ Vue2
    • https://v3.cn.vuejs.org/ Vue3
  • 是中国的⼤神尤⾬溪开发的，为数不多的国⼈开发的世界顶级开源软件
  • 是⼀套⽤于构建⽤户界⾯的渐进式框架，Vue 被设计为可以⾃底向上逐层应⽤。
  • MVVM 响应式编程模型，避免直接操作 DOM , 降低 DOM 操作的复杂性。

  • MVVM：⻚⾯输⼊改变数据，数据改变影响⻚⾯数据展示与渲染
    • M（model）：普通的 javascript 数据对象
    • V（view） ：前端展示⻚⾯
    • VM（ViewModel）：⽤于双向绑定数据与⻚⾯，对于我们来说，就是 Vue 的实例

2. Vue 的核⼼功能

  • 基础功能：⻚⾯渲染、表单处理提交、帮我们管理 DOM(虚拟 DOM)节点。

  • 组件化开发：增强代码的复⽤能⼒，复杂系统代码维护更简单。

  • 前端路由：更流畅的的⽤户体验、灵活的在⻚⾯切换已渲染组件的显示，不需与后端做多余的交互。
  • 状态集中管理：MVVM 响应式模型基础上实现多组件之间的状态数据同步与管理。
  • 前端⼯程化：结合 webpack 等前端打包⼯具，管理多种静态资源，代码，测试，发布等，整合前端⼤型项⽬。

3. 渐进式框架

“渐进式框架”的理解，就是⽤你想⽤或者能⽤的功能特性，你不想⽤的部分功能可以先不⽤。

Vue 不强求你⼀次性接受并使⽤它的全部功能特性。

  1. 场景⼀：公司刚开始⼀个项⽬，技术⼈员对 Vue 的掌握也不⾜够。那么我们就不能使⽤ Vue 了么？当然不是，如果你只是使⽤ Vue 做些基础操作，如：⻚⾯渲染、表单处理提交功能，那还 是⾮常简单的，成熟的技术⼈员上⼿也就⼀两天。完全可以⽤它去代替 jQuery ，并不需要你去引⼊其他复杂特性功能。
  2. 场景⼆：我们的项⽬⽤了 VUE ，使⽤的效果也挺好。现在我们想逐渐实现代码组件化，实现代码的复⽤，或者是基于组件原型的跨项⽬的代码复⽤，那么可以引⼊ Vue 的 components 组件特性就可以了。
  3. 场景三：当项⽬规模逐渐变⼤了，我们可能会逐渐⽤到前端路由、状态集中管理、并最终实现⼀个⾼度⼯程化的前端项⽬，这些功能特性可以逐步引⼊。

所以 Vue 的适⽤⾯很⼴，你可以⽤它代替⽼项⽬中的 jQuery ，也可以在新项⽬启动初期，有限地使⽤ Vue 的功能特性，从⽽降低上⼿的成本。

三、理解 MVVM 模型

1. MVVM 模型

之前，在前端响应式框架 Vue 简介中我们曾简单介绍过 MVVM 模型。

  • MVVM：⻚⾯输⼊改变数据，数据改变影响⻚⾯数据展示与渲染
    • M（model）：普通的 javascript 数据对象
    • V（view）：前端展示⻚⾯或前端组件
    • VM（ViewModel）：⽤于双向绑定数据与⻚⾯，对于我们来说，就是 Vue 的实例

具体到编码层⾯，注意我们提到的 MVVM 都是前端的概念，体现如下：

  • ViewModel（ Vue 的实例）作为 MVVM 模型的核⼼，负责监听 DOM 的变化去修改 Model（data）数据；负责将数据绑定到⻚⾯上进⾏数据展示及⻚⾯渲染；是 View 与 Model 沟通的桥梁。
  • ViewModel 帮我们实现了双向数据绑定的底层⽀撑，是 Vue 帮我们实现的。
  • ViewModel 是对视图及模型的抽象，即可以操作视图，也可以操作数据模型，并实现⼆者的⾃动响应。

2. MVVM 模型与直接操作 DOM 的区别

我们来使⽤ JS 操作 DOM 的⽅式完成上⾯的代码完成的⼯作。

⾸先，我们需要先选取需要进⾏操作的 DOM 节点，为此我们需要为 h2 标签加上⼀个 id ，然后通过 ajax 向服务器发送请求，接收服务端的数据，通过 DOM 节点，操作节点的数据和显示样式。

let dom = document.getElementById("h2id");

//向服务器发送ajax请求获取数据，如 '你好，Vue3';

dom.innerText = "你好，Vue3";

然⽽ MVVM 编程模型不需要关⼼ DOM 的结构，同样的操作只需要下⾯这⼀⾏代码：

app.message = "你好，Vue3";
  • MVVM 模型不关⼼ DOM 的结构，不需要寻找 DOM 节点
  • MVVM 模型通过修改模型数据改变视图显示，不使⽤ DOM 的 innerText、innerHtml 等
  • MVVM 模型要求我们在前端考虑数据模型，这样有更利于前后端解耦，实现前后端分离应⽤的开发。（也就是说，开发阶段可以先在前端模型⾥⾯写假数据，前后端并⾏开发）

3. MVVM 和 MVC 模式区别与联系

  • MVVM 是 MVC 模式的⼀个演变，是 John Gossman 在《Model/View/ViewModel pattern for building WPF apps》中提出的。
  • 在实际的开发应⽤中，MVC 是后端开发⼴泛应⽤的模式，⽽ MVVM 模式是前端⼴泛应⽤的模式，从使⽤的⻆度⼆者并⽆交集。建议：不要将⼆者强⾏关联⽐较，学 MVC 的时候学 MVC（后端），学 MVVM 的时候就学 MVVM（前端），这样就⾜够了。
  • MVC 的 Model 指的是服务端的 Model ，⽽ MVVM 的 Model 指的是 javascript 端的 Model 。这个变化⾮常有意义，真正的实现了前后端的分离。前端开发可以先在 model ⾥⾯写假数据，然后再与后端进⾏数据接⼝对接。
  • ⼆者的核⼼思想都是将视图（View）与模型（Model）解耦，即视图与数据的解耦。

随着后续通过实际的应⽤的开发，相信⼤家会对 MVVM 模式有更加深⼊的理解。

四、实现字符串反转和内容隐藏功能

1. 字符串翻转

代码示例

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta http-equiv="X-UA-Compatible" content="IE=edge" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Vue3实现字符串的反转</title>
  </head>
  <body>
    <div id="root"></div>
    <script src="https://unpkg.com/vue@next"></script>
    <script>
      Vue.createApp({
        data() {
          return {
            content: "hello world",
          };
        },
        methods: {
          handleBtnClick() {
            // 打散成数组、逆转数组、拼接成字符串
            this.content = this.content.split("").reverse().join("");
          },
        },
        template: `
          <div>
            <p>{{content}}</p>
            <button v-on:click="handleBtnClick">反转</button>
          </div>  
        `,
      }).mount("#root");
    </script>
  </body>
</html>

说明：

  1. v-on 绑定事件，v-on:click 绑定点击事件
  2. click =…… 执⾏⼀个什么样的函数
  3. 函数定义在哪⾥？定义在 methods()⾥⾯
  4. split(‘’).reverse(‘’).join(‘’) ：字符串打散成数组——反转——再连接为字符串

运⾏可以实现字符串的反转

2. 内容隐藏

代码示例

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta http-equiv="X-UA-Compatible" content="IE=edge" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Vue3实现内容隐藏</title>
  </head>
  <body>
    <div id="root"></div>
    <script src="https://unpkg.com/vue@next"></script>
    <script>
      Vue.createApp({
        data() {
          return {
            show: true,
          };
        },
        methods: {
          handleBtnClick() {
            this.show = !this.show;
          },
        },
        template: `
        <div>
          <p v-if="show">hello world</p>
          <button v-on:click="handleBtnClick">显示/隐藏</button>
        </div>
      `,
      }).mount("#root");
    </script>
  </body>
</html>

说明：

  1. v-if 显示/隐藏，v-if=”show” <p>标签的存在与否取决于 show 这个变量的值
  2. methods 中对 data 中的数据进⾏读取，必须使⽤ this，⽐如 this.show = !this.show ;否则就⽆法读 取，当然也可以写全，⽐如 this.$data.show = ……
  3. 隐藏展示，可以做代码优化
    • v-if : 条件渲染
    • v-show :已经渲染

运⾏可以实现内容的显示和隐藏

五、编写⼀个简单的 TodoList，了解循环和双向绑定

代码示例

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Vue3实现todolist</title>
  </head>
  <body>
    <div id="root"></div>
    <script src="https://unpkg.com/vue@next"></script>
    <script>
      Vue.createApp({
        data() {
          return {
            inputValue: "",
            list: [],
          };
        },
        methods: {
          handleAddItem() {
            this.list.push(this.inputValue);
            this.inputValue = "";
          },
        },
        template: `
        <div>
          <input v-model="inputValue" />
          <button v-on:click="handleAddItem">增加</button>
          <ul>
            <li v-for="(item, index) of list">{{item}}</li>
          </ul>
        </div>
      `,
      }).mount("#root");
    </script>
  </body>
</html>

说明：

1、v-for 指令：对数组内容进⾏循环

  • v-for=’item of list’ 通过 v-for 这个指令循环 list ⾥⾯的数据，把每⼀个数据放到 item ⾥⾯去
  • v-for=’(item,index) of list’ 循环 list ⾥⾯的数据，把 list ⾥⾯的内容放到 item ⾥⾯，把循环到第⼏项的下标放到 index ⾥⾯

2、v-model 指令：可以让数据和 input 输⼊框双向绑定

input ⾥⾯输⼊和内容和 data 中的 inputValue 变量这样绑定，它们两个是双向绑定的关系，⼀个发⽣变化另⼀个就会发⽣变化。

六、组件初探：对 TodoList 进⾏组件代码拆分

代码示例

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta http-equiv="X-UA-Compatible" content="IE=edge" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>vue3-todolist组件化</title>
  </head>
  <body>
    <div id="app"></div>
    <script src="https://unpkg.com/vue@next"></script>
    <script>
      // mvvm、vue实例，vue组件
      const app = Vue.createApp({
        data() {
          return {
            inputValue: "",
            list: [],
          };
        },
        methods: {
          handleAddItem() {
            this.list.push(this.inputValue);
            this.inputValue = "";
          },
        },
        template: `
          <div>
            <input v-model="inputValue">
            <button v-on:click="handleAddItem" v-bind:title="inputValue">
              增加
            </button>
            <ul>
              <todo-item v-for="(item,index) of list"
              v-bind:content="item"
              v-bind:index="index"
              />        
            </ul>  
          </div>
         `,
      });
      app.component("todo-item", {
        props: ["content", "index"],
        template: "<li>{{index}}——{{content}}</li>",
      });
      app.mount("#app");
    </script>
  </body>
</html>

说明：

  • 插值表达式{{}}
  • 注册组件 component
  • v-bind 和 props ⼀起使⽤，实现组件传参
  • v-bind 把数据与标签的属性绑定

v-bind:title, ⽤ v-bind 绑定某个标签属性

对于较⼤的项⽬，可以将某些较⼩的粒度拆分后形成组件，并进⾏注册。

app.component( <componentName>, {<content>};

要注意注册组件的先后顺序。

可以先创建⼀个 app，注册完所有组件后，再将 app 挂载到某个节点下。

如果对某个标签⽤ v-bind 进⾏了属性的绑定，可以在组件中采⽤ props 来获取这些属性的值，可以⽤ 多个 v-bind 绑定多个属性。

如果组件名为 “todo-list” ，标签中可以直接写<todo-list/>，Vue 能够识别
